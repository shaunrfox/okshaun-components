  # Size Tokens

  Size tokens define consistent size values used throughout the design system for width, height, borders, border-radius, margins, padding, gaps, etc.

  We use a type-based scale to size everything in the design system, and all size tokens are defined in `rem` units (rem = root em font size). This means that the size of the font drives the size of all other elements. The scale is generally divisible by 8 or 4, with the exception of some reserved utility tokens for specific use cases.

  ### Container Sizes

  These `containerSizes` are used for conditional styling using CSS Container Queries. <Link href="https://panda-css.com/docs/concepts/conditional-styles#container-queries" target="_blank" external>Using container queries in Panda CSS</Link>
  
  CSS Container Queries are a modern web design feature that allows elements to adjust their styles based on the size or other characteristics of their parent container, rather than the overall browser viewport size. This enables the creation of highly reusable, flexible components that adapt to where they are placed in any layout.

  ### Breakpoints

  These `breakpoints` are used in media queries to  conditionally change the styles of the element based on the size of the viewport. <Link href="https://panda-css.com/docs/docs/concepts/responsive-design" target="_blank" external>Using breakpoints in Panda CSS</Link>

  CSS media queries are a foundational feature of responsive web design that allow developers to apply styles based on the characteristics of the user's device or viewing environment, such as screen size, resolution, and orientation.
